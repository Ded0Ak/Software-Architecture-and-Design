# Software Architecture & Design — Lab 4 (WebUI)

## Опис завдання
Реалізовано WebAPI для управління **послугами, портфоліо та замовленнями** дизайн‑студії з рівнем представлення (UI).  
Система складається з шарів:
- **UI (HTML/JS)** — інтерфейс користувача
- **API (Express)** — приймає HTTP‑запити
- **BLL** — бізнес‑логіка
- **DAL** — збереження у JSON‑файл
- **InMemoryDbContext** — робота з даними в памʼяті з автозбереженням

### Реалізовано:
- CRUD для **Services / Portfolio / Orders**
- Перевірки на унікальність
- Автододавання замовлень у портфоліо
- Синхронізація портфоліо при зміні/видаленні замовлення
- PUT/DELETE
- UI кнопки для редагування/видалення
- Розділення за ролями (UI/API/BLL/DAL)
- Генерація окремих ID для кожної сутності

# Відповіді на контрольні питання

### 1. Що таке API? Що таке WebAPI?
**API** — інтерфейс для взаємодії між програмами.  
**WebAPI** — API через HTTP/HTTPS, яке повертає дані у форматі JSON/XML.

### 2. Які задачі вирішує та в чому особливості рівня представлення PL?
PL (Presentation Layer) показує дані користувачу та приймає взаємодію. Його задача — **відображення, UI/UX, валідація вводу**, мінімум бізнес‑логіки.

### 3. Які типові шаблони проектування існують для реалізації рівня представлення PL?
MVC, MVP, MVVM, PAC, HMVC, MVU.

### 4. В чому особливість та як реалізується шаблон MVC?
MVC розділяє **Model, View, Controller**.  
Controller обробляє запит, викликає Model, передає дані у View.

### 5. В чому особливість та як реалізується шаблон MVP?
MVP: View пасивна, Presenter містить всю логіку та спілкується з View через інтерфейс.

### 6. В чому особливість та як реалізується шаблон MVVM?
MVVM: ViewModel містить стан і логіку, View зв’язана через data binding.

### 7. Опишіть шлях обробки запиту (MVC).
Браузер → Router → Controller → Model → Controller → View → HTML → Браузер.

### 8. Чим відрізняється ASP.NET MVC та ASP.NET WebAPI?
MVC повертає View (HTML), WebAPI повертає дані (JSON/XML).

### 9. Відмінність WebAPI від інших клієнт‑серверних застосувань?
WebAPI не має UI, повертає дані, працює як сервіс.

### 10. Які види серверів ви знаєте?
Web server, Application server, Database server, File server, Proxy, Mail server.

### 11. Що таке веб сервер і його задачі? Приклади.
Обробляє HTTP, віддає статику.  
Приклади: **Nginx, Apache, IIS**.

### 12. Що таке сервер застосувань?
Виконує бізнес‑логіку та обробляє запити.  
Приклади: **Node.js, .NET Kestrel, Tomcat**.

### 13. Поясніть протокол HTTP.
HTTP — протокол запит/відповідь.  
Клієнт надсилає метод + URI + headers, сервер повертає статус + тіло.

### 14. Різниця GET/POST/PUT/DELETE
GET — читання  
POST — створення  
PUT — повне оновлення  
DELETE — видалення

### 15. Групи статус кодів HTTP
1xx — інформаційні  
2xx — успіх  
3xx — редиректи  
4xx — помилки клієнта  
5xx — помилки сервера

### 16. Особливість DI у ASP.NET MVC/WebAPI
Контейнер інжектить залежності в контролери через конструктор, життєвий цикл керується контейнером.

### 17. Postman / Fiddler
**Postman** тестує API запити.  
**Fiddler** перехоплює HTTP‑трафік для аналізу.

### 18. Різниця між веб застосуванням та веб сервісом
Веб застосунок має UI, сервіс — тільки API/дані.

### 19. Що таке REST?
Архітектурний стиль для WebAPI, де ресурси адресуються URL.

### 20. REST vs RPC vs SOAP
REST — ресурси + HTTP.  
RPC — виклик методів.  
SOAP — XML + строгий контракт.

### 21. Маршрутизація ASP.NET WebAPI
Маршрут визначає URL → Controller → Action через атрибути або шаблон.

### 22. Як визначається контроллер/метод?
По маршруту + HTTP методу (GET/POST/PUT/DELETE) + назві action.
